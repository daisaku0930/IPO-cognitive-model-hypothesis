# IPO認知モデル ― 原典（通し原稿）

人間のあらゆる認知や営みを **Input → Process → Output** として捉え直し、その個人差を座標で扱う観察モデル。本ファイルは、比喩や噛み砕きを外して構造だけを一本に通した記録である。読み物としての説明は本編、定義の最終確定は正典、操作手順は使い方が持つ。

> 位置づけ：これは「理論」ではなく観察フレームワーク。構造（軸）の上に、外れうる理論（軸どうしのリンク・法則）が薄く乗る。形式化（記号がある）はされているが操作化（測定手段がある）はされていない。

---

## 0. 式（モデルの署名と状態の定義）

```
IPO × C

C = (W, n, m | α, Θ, Δ)
```

- 1行目 **`IPO × C`** … モデルの署名。**流れ（IPO）× 状態（C）** の掛け合わせ。IPO は「どこから・どの向きで読むか」（起点の軸）、C は「いまの認知配置」（状態の軸）。両者は階層が異なるので行で分ける。
- 2行目 **`C = (W, n, m | α, Θ, Δ)`** … 状態 C の定義。縦棒の左（W・n・m）＝状態変数、右（α・Θ・Δ）＝補正パラメータ（すべてP層）。
- **時間性**：C は固定値ではなく、IPOサイクルを通じて更新される状態（C_t → C_t+1）。更新は§6。
- **補助記法**：O_ext＝外に出す結果（他者が見る）／O_int＝内に残る結果（次サイクルのCに混ざる）。Cの成分ではなく、Output界面の二方向を区別する記法。

## 1. 骨格 ― IPOと3つの志向

すべての認知を `Input → Process → Output` の流れとして見る。同じ流れを見ても、どこに価値・関心を置くかが異なる。

| 志向 | 関心 | 主な問い |
| --- | --- | --- |
| I（Input） | 情報・材料・前提・根拠 | 情報は揃っているか／根拠は／他に事例は |
| P（Process） | 判断・構造・因果・仕組み | なぜそうなるか／どう判断するか／応用できるか |
| O（Output） | 結果・成果・正解 | 結局どうなったか／正解は／どこまでやるか |

これは固定の「型分け」ではない。後述する W（注意配分の方向）の偏りの典型例にすぎず、**W は人に固定された属性ではなく場面によって振れる**。同じ人が監査の場では I、企画の場では O に寄りうる。「I型／P型／O型」は『その場面でその志向に寄っている状態』の略記である。【仮説】

> 「→」自体もProcessである。`I → P → O` の P 内部は `I → P1 → P2 → … → O` の多段構造をもつ。この刻みの深さが後の m（§4）。

---

## 2. W ― 注意配分の方向

「どこに注意を向けるか」を表す重みベクトル。

```
W = (w_I, w_P, w_O)   （Σw = 1）
```

W は方向のみを表す。これと独立に「どこまで細かく扱うか」（解像度 n、§3）と「どこまで深く掘るか」（深度 m、§4）の軸がある。【仮説】

すれ違いの一部は、能力差ではなく W の差で説明できる。たとえば「答えを教えてほしい」（O寄り）と「考えればわかるはず」（P寄り）は、どちらも正しく、見ている場所が違うだけである。【仮説】

---

## 3. n ― 解像度（界面・並列の広さ）

n はスカラーではなく、I・P・O に対応する3成分のベクトル。

```
n = (n_i, n_p, n_o)
```

- **n_i**：入力の解像度（世界からどれだけ細かく受け取るか／観察・傾聴の粒度）
- **n_p**：Pの並列度（同時に保持できる処理スレッド数／ワーキングメモリ的な幅）
- **n_o**：出力の解像度（世界へどれだけ細かく出すか／表現の粒度）

**界面はシングルスレッド**：`I → P → O` のパイプライン自体は並列に流せない（「人はマルチタスクをしない」と言われる所以）。並列化できるのは P 内部のみで、その幅が n_p。ワーキングメモリの容量限界（いわゆる 7±2）は n_p の上限の現れと解釈できる。【確立：n_p ↔ WM容量】

**n_i と n_o は独立に異なりうる**（旧モデルが n_i = n_p = n_o を暗黙に仮定していたことの修正）。【仮説：二重乖離として検証可能】

- n_i 高・n_o 低：細かく受け取り粗く出す（観察者・研究者）
- n_i 低・n_o 高：粗く受け取り細かく出す（表現者・職人）
- n_i 高・n_o 高：両方細かい（翻訳者・通訳）

**会話の噛み合わなさ**：「少し考えればわかるはず」が通じないのは、自分（n_p大）に自明な処理の刻みが、相手（n_p小）の認知構造にそもそも存在しないため。能力ではなく構造の問題。差の出どころが n_i／n_p／n_o のどれかで、すれ違いの種類が変わる。【仮説】

> 関数合成による式化（補足）：流れは `O = (f_n ∘ … ∘ f_1)(I)` と書ける（f_k は解釈・分類・判断・行動選択などの変換）。これは記述の便宜であり、測定された量ではない。

## 4. m ― 処理の深さ

P 内部で生成された中間成果物が、さらなる処理の入力として再投入される段数。n が「横の広さ」なら m は「縦の深さ」。

```
m 小（浅い）：I → P → O
m 大（深い）：I → P → Pk → P' → … → O   ← 中間結果が入力として再帰
```

- **Pk・I' は相の名前**：同一の実体が「P内部の中間成果物(Pk)」とも「再投入される材料(I')」とも呼べる。物が変わるのではなく、流れの中の局面（相）が変わる呼称。この「中間成果物が入力へ転じる臨界点」を、本モデルは**相転移（処理の臨界＝デジタルな切れ目）**として扱う（他の切れ目との並びは§8）。
- **連鎖は深さの外部展開**：O を新たな I として再投入する「連鎖」は、深さ m が1ユニットに収まらず O界面を経由して展開されたもの。内部で完結すれば深い処理、外部（O_ext）を経由すれば連鎖に見える。同じ深化の、界面をまたぐか否かの差。
- **n_p が m の上限を縛る**：並列に保持できる中間成果物が少なければ、深い再投入は起動しにくい。【仮説】
- **深い処理が起きない条件**：n_p が強制的に小さくされる（マニュアル化・ルーティン化で並列保持を許さない）／O の手前で止める（O型傾向）／時間・リソース不足。m が浅く固定されると深化が起こらない。これは「なぜ定型業務で学習・創造が起きにくいか」の構造的説明。【仮説】

**射程の拡張**：m を加えると、本モデルは認知タイプの記述を超え、学習（中間成果が新たな問いを生む）・問題解決（部分解がより深い問題を開く）・創造（断片が別の断片を誘発する）の発生条件を説明する側へ広がる。

> 検討中の再定義：m を「Process内部の深さ」ではなく、**I/P・P/O・P/P' の3界面をまたぐ往復回数**として読み直せないか。n（界面の広さ）と m（界面の往復）が同じ土俵に乗り、どの往復にも P が噛むため m が P 専属である理由も導ける。ただし観測できるのは I/P・P/O 往復のみで、P/P'（黙考・内省）は外から見えず、統一と引き換えに測定窓は狭まる。【検討課題】

## 5. 補正 ― α・Θ・Δ（伝達と自己補正の層）

認知は、明示されていない行間を自動で補う（補正）。補正はすべて P 層で起こり、界面（I・O）は構造的制約として補正対象外。深さ m に作用する。

同じ入力でも着地は3段階に分かれる。

| 段階 | 状態 |
| --- | --- |
| ① 快適 | 自動で埋まる（無自覚） |
| ② コスト増 | 埋まるが意識的努力が要る |
| ③ 破綻 | 埋めきれない（誤読・フリーズ・回避） |

これを決める3パラメータは **α → Θ → Δ** の階層をなす。

- **α（発火確率）**：補正がそもそも実行されるか。低いと埋める前に読み飛ばす＝うっかりミス（能力でなく発火の問題）。
- **Θ（閾値）**：自動で埋められる範囲。①↔②の境。
- **Δ（容量）**：埋められる力の上限。②↔③の境。

完全構造化文書（結論・根拠・結果のみ）を高解像度の人が「読みやすいが読みにくい」と感じるのは、Δ超え（理解不能）ではなく Θ超え（理解可能だが補正コスト）＝段階②だからである。【仮説】

> 但し書き（この層は最も思弁的）：α・Θ・Δ には測定手段がなく、どんな失敗も事後に「α低下」「Δ超過」で説明できてしまう。意味を持つのは、3段階が別々の観察可能なサイン（即時無自覚／遅延・自覚・疲労／誤り・回避）として分離できるときだけ。そこまで絞れば【仮説】、絞る前は【思弁】。

### 変数構造の総括

骨格が I・P・O の3要素である以上、各変数は3軸に沿った成分を持ちうる。

- **W・n**：I/P/O に対応する3成分の**ベクトル**（方向／広さ）。
- **m・α・Θ・Δ**：P に集約された**スカラー**（深さと、その深さに作用する補正）。

## 6. 更新 ― C の自己更新

C は固定された認知特性ではなく、IPOサイクルの経験を通じて更新され続ける（C_t → C_t+1）。【思弁】

1回のサイクルは、外に出す結果（O_ext）だけでなく、内に残る結果（O_int）を生む。O_int が次サイクルの C に混ざることで C が変わる。

**更新の向きは経験そのもの（O_ext）でなく、間に挟まる解釈（P）で決まる**。同じ結果でも「自分は向いてない」（W→自己評価）と「この手順の穴だ」（W→構造）で、残る O_int が違い、更新方向が分岐する。ゆえに「経験を積めば誰でも伸びる」は成り立たず、伸ばすのは O_ext の量でなく解釈 P の質。【仮説】

更新の波及：n_p が空く（チャンク化【確立】）→ m が深く掘れる → Θ が広がり段階②が①になる（「楽になった」）→ W が動く。逆に O_int が残らなければ（ルーティンで m 浅化）、回数だけ増えて C は更新されない。

この構造をとるとき「個」とは、固定属性の束ではなく、**世界から Input を受け取り Output を返しながら自己を更新し続ける再帰システム**として定義される。これは「なぜ人は変わるのか」の構造的説明でもある。

> 但し書き：これも事後的に何でも説明しうる。検証可能になるのは、同一 O_ext から異なる O_int が分離して観察できるときだけ。【仮説】

## 7. 起点 ― IPO無限連続とOPI

「→」も Process であり、O や Pk は新たな I として再投入される。両者を認めると、認知は単発の IPO ではなく無限連続として表せる。

```
… → I → P → O → I → P → O → …
```

この連続の上では**起点は固定されていない**。どこから読み始めるかは観測者・本人の取り方の問題。I から読めば IPO、O から読めば OPI。

- **OPI**：O（先行サイクルの O_int が確信として浮上した点）を起点に体験・記述するパターン。別の能力ではなく、同じ連続を違う起点から読んでいるだけ。
- **創造 ＝ 起点を選ぶ自由度**：O を起点に取り、後から P（言語化）と I（素材調達）で支える。起点固定（常に I から）なら与えられた材料の組み合わせしか出ない。
- **「直感型」に見える2種**：OPI型（確信が先・素材が後）と、超高速IPO型（起点は I だが処理が速く直感に見える）。内側の順序は違うが、**外からも本人からも識別する操作的定義は現状ない**。

> 但し書き：起点の自由度は外から判別できず（超高速IPO型とOPI型が区別不能）、反証可能性はほぼゼロ。ここは「測る主張」ではなく「視点を増やす装置」。価値は予測力でなく生成力。【思弁】

## 8. クオリア（帰属としての扱い）

3つの変換面を導入する：知覚面（I→P）・表出面（P→O）・社会的境界面（A↔B）。

クオリア（主観的な感じ）は、観測者 B が、相手 A の境界面に現れた信号から **構成する帰属** として記述される。人間が AI の出力から内側を逆推定し主観を帰属する現象も、対人間とまったく同じ発生機構を持つ。本モデルは帰属の**発生機構の同一性**を述べるのみで、帰属の**妥当性**（実際に内的状態があるか）は判断しない。【思弁】

**境界面の存在論的地位**：I→P・P→O の変換点は連続に引いた観測格子の線だが、**A↔B の社会的境界面だけは観測者が引いた線ではなく物理的に実在する分断**（二つの認知系は別個の系）。本モデルがデジタルな切れ目として扱う点は3つ：相転移（§4）・Θ越え（§5）・個体間境界（本節）。

> スコープ（限定）：説明するのは帰属がどう発生するか（他我問題の側）であって、なぜ感じが伴うか（意識のハードプロブレム）ではない。一人称クオリアは観測不能であり、本モデルの外部。**モデル内でクオリアは常に「帰属」としてのみ登場する。**【思弁】

## 10. 検討課題（要約）

最大の宿題は**操作的定義**：W・m・α・Θ・Δ に測定手段（代理指標）を作れるか。ほかに——

- 型か状態か（W を場面依存の連続分布として扱うべきか）／I・P・O は実在するか・分布は
- n_i・n_p・n_o は独立か相関するか／n_p の上限は WM 容量と一致するか
- m の界面往復による再定義（I/P・P/O・P/P'）は妥当か／深い処理の発生条件をより精密にできるか
- α・Θ・Δ は独立した3パラメータか／Δ は単一能力か作用点ごとの束か（二重乖離は観察されるか）／Θ はヒステリシスを持つか
- C はサイクルで自己更新されるか／同一経験から更新方向が分岐する条件は何か
- クオリア帰属の発生条件／人間への帰属と AI への帰属は構造的に同一か
- OPI型と超高速IPO型を識別する操作的定義は作れるか
- **既存理論との差分**：目標志向性・処理水準・WM容量・構成レベル理論と、測って差が出るか（出なければ再記述に留まる ← 正念場）

> 評価方針：本モデルは高抽象度の解釈モデルであり、評価は反証可能性のみに依らず、主観的照合や説明力も判断材料となる。ただし「何でも説明できる説明は何も予測しない」を常に重しとする。適用範囲は人間が認知・観測可能な領域に限定し、その外は本モデルの外部として扱う。

## 11. 発展の記録（A：出発点との差分）

本原典は「IPO認知モデル仮説_加筆版-4」を出発点とする。主な発展は以下。

- **n の再定義**：当初 n を「Processの分解数」という単一スカラーとしていた。これはモデルの作者が P型であったため P の解像度だけを高く見た**射影**だった。I・P・O それぞれに解像度を持つベクトル `(n_i, n_p, n_o)` へ再定義。旧定義の分解数は n_p（および m へ移譲した深度）に相当する。**モデルの形が作者の C を反映していた**という気づきが、全変数が3軸成分を持ちうるという総括（§5）につながった。
- **m の再定義**：当初「IPOユニットの連鎖数」（横の繰り返し）としていたが、n を広さと再定義したのに伴い m を「処理の深さ」（縦）へ整理。連鎖は深さの外部展開として統合（§4）。
- **補正層の分離**：状態変数（W・n・m）と補正パラメータ（α・Θ・Δ）を階層として分離し、式を `C = (W, n, m | α, Θ, Δ)` に。さらに署名 `IPO × C` を上位に置いた（§0）。
- **時間性の明示・補助記法**：C_t → C_t+1 の自己更新と、O_ext／O_int の補助記法を追加（§0・§6）。
- **削除した枝**：学習スタイル論との優劣比較、および哲学的ゾンビ問題への接続。前者は本筋に不要、後者はゾンビ論証の前提を受け入れているだけで解決でも言い換えでもないため、原典からは外した。クオリアは「帰属」としてのみ残す（§8）。
