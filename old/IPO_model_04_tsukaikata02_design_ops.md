# 使い方02 ― 設計と運用

使い方01は「**一回の出力**を相手の C に合わせる」話だった。
ここでは時間幅を広げる。**設計**＝他者の C が育つ場を長期でつくる。**運用**＝自分の C をその場その場で操作する。どちらも同じ座標を、別の時間幅で扱っている。

---

# 設計軸 ― 学びが起きる場をつくる

## マニュアルは、たいていO層しかない

普通のマニュアルは**手順（O）**だけが書いてある。
すると I型は「根拠は？適用範囲は？」で止まり、P型は「なぜこの順番？」で止まる。
3層を補えば、どの型にも入口ができる。

| 層 | 補う内容 | 効く相手 |
| --- | --- | --- |
| I層 | 前提・材料・適用範囲・例外条件 | I型 |
| P層 | なぜこの手順か・判断の分岐点 | P型 |
| O層 | 手順そのもの（既存） | O型 |

## Process は渡せない、足場は渡せる

他人の Process を直接操作することはできない。
だが、低 n_p の読み手に欠けている「**間の刻み**」を、足場として置くことはできる。
本編02の、A → D しかない新人に B・C を差し込む――これが足場かけだ。【仮説】

## 設計のジレンマ ― 全部書くと、学ばない

ここで本編02の裏返しが効く。
**全部を spell out すると、読み手は行間を埋める必要がなくなる**＝深さ m が発火しない＝学習が起きない（ルーティン化とまったく同じ構造）。
逆に、足りなすぎると Δ を超えて破綻する。

```
全部書く（Θの内）   → 即できる、でも育たない
少し余白（Θの少し上）→ 自分で埋める、育つ      ← 狙い
余白すぎ（Δの外）    → 埋まらない、折れる
```

狙いは「**Θ の少し上、Δ の下**」。読み手が自分で少しだけ埋める余地を残す。これが「ちょうどいい難しさ」だ。【仮説】

> 目盛りの使い分け：**即戦力マニュアル**は Θ の内（迷わせない）。**育成教材**は Θ の少し上（あえて考えさせる）。目的で位置を変える。

# 運用軸 ― 自分の C をその場で操作する

## C はツマミである（コックピット）

C は固定値の記述であると同時に、回せるツマミだった（本編04・05）。
状況に応じて**どれを回すか**を選ぶのが運用だ。

| 場面 | 上げる | 下げる |
| --- | --- | --- |
| レビュー・監査 | m（掘り下げ）・n_i（読み込み） | スピード要求 |
| 提出・報告 | W=O（結論優先） | m・n_o（細かさを絞る） |
| 着想・企画 | α（探索の幅） | 正確さの要求 |

## Temperature論 ― α を意図的に振る

α は本来「補正の発火」だが、運用では「**探索の幅**」として振れる。

- **高温**（α のばらつき大）：発散・着想。突飛な接続を許す
- **低温**（α が安定）：収束・実行。毎回同じ品質を出す

フェーズで温度を変える。**着想は高温 → 実行は低温**。
AIの temperature とまったく同じ発想で、人にもチームにも適用できる。【思弁】

## 「完璧な最適化」は誤り

すべてのツマミを最大にするのは、最適ではない。

- m を常に最大 → 分析麻痺、いつまでも終わらない
- n を常に最大 → 過負荷、何も出てこない
- 常に高温 → 永遠に出荷しない

**グローバルに最適な C は存在しない。** あるのは、場面ごとに適切な C だけだ。
だから「全部を70点で素早く回し、要所だけ深掘る」ほうが、全項目満点を狙うより強いことが多い。【仮説】

# まとめ ― 同じCを、3つの時間幅で

| 何を | 対象 | 時間幅 |
| --- | --- | --- |
| 使い方01 | 一回の出力 | 瞬間 |
| 設計 | 他者の C が育つ場 | 長期 |
| 運用 | 自分の C のツマミ | リアルタイム |

すべて同じ `C = (W, n, m | α, Θ, Δ)` を扱っている。違うのは、それを読むのか・書くのか・育てるのか・回すのかだけ。定義は**正典**を参照。
