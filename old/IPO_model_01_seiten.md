# IPO認知モデル ― 正典（リファレンスコア）

> このブロックはモデルの**単一の定義元**。ほかの節はここを参照するだけで、再定義しない。
> 人間は上から普通に読める。AIは front matter の `ai_usage` を先に読む。

---

## 1. 骨格

人のあらゆる認知を **Input → Process → Output** として捉え直す観察モデル。
同じI→P→Oを見ても、どこに価値・興味を置くかが人によって違う、という出発点に立つ。

- **I型（Input注視）** … 情報・根拠・前提が揃っているかを気にする
- **P型（Process注視）** … なぜそうなるか・どう判断するかを気にする
- **O型（Output注視）** … 結局どうなったか・正解は何かを気にする

これは「型分け」ではなく、後述するWの偏りの典型例にすぎない。
さらに、W は人に固定された属性ではなく**場面によって振れる**。同じ人が、監査の場では I、企画の場では O に寄ることがある。だから本書の「I型／P型／O型」は『その場面でその志向に寄っている状態』の略記であって、人につける固定ラベルではない。（特性か状態かの掘り下げは付録A「型か状態か」へ。）

---

## 2. 正典の式

```
IPO × C

C = (W, n, m | α, Θ, Δ)
```

式は2段に分かれている。**階層の違う2つ**を、行で分けて示している。

- 1行目 **`IPO × C`** … モデルの署名。**流れ（IPO）× 状態（C）** の掛け合わせ。
  - **IPO** ＝ 流れの軸：この認知を「どこから・どの向きで読むか」（→ 本編05）
  - **C** ＝ 状態の軸：いまの認知配置
  - **×** ＝ 階層の違う2つを掛け合わせる。認知の全体像は「流れ × 状態」として表れる
- 2行目 **`C = (W, n, m | α, Θ, Δ)`** … 状態 C の**定義**。`=` は素直に「C の内訳」を意味する。
  - 縦棒の **左（W, n, m）= 状態変数**：いまの認知状態を表す
  - 縦棒の **右（α, Θ, Δ）= 補正パラメータ**：状態に作用する能力・閾値。すべてP層に属する

> 署名（IPO × C）と定義（C = …）を1行に押し込まないのは、IPO と C が同じ階層の量ではないため。流れの上に状態が乗る、という階層差を表記でも保つ。

## 3. 変数表（唯一の定義元）

| 記号  | 名称（AI言い換え）     | 意味                 | 構造                           |
| --- | -------------- | ------------------ | ---------------------------- |
| W   | 注意の向け先         | どこに注意を配分するか        | ベクトル `(w_I, w_P, w_O)`, Σw=1 |
| n   | 受け取り・並列・出力の細かさ | 界面と並列の解像度（横の広さ）    | ベクトル `(n_i, n_p, n_o)`       |
| m   | 掘り下げの段数        | Pの処理の深さ（縦の深さ・再帰段数） | スカラー（P専属）                    |
| α   | 補正が実行される確実さ    | 補正がどれだけ確実に発火するか    | スカラー（P専属・mに作用）               |
| Θ   | 無意識に補正できる範囲    | 補正が自覚に上がる閾値        | スカラー（P専属・mに作用）               |
| Δ   | 行間の補正力の上限      | 補正がどこまで可能か         | スカラー（P専属・mに作用）               |

**nの内訳**：n_i = 受け取りの細かさ／n_p = 同時に保持できる処理の本数（≒ワーキングメモリ幅）／n_o = 出す細かさ。
**接続関係**：n_p の大きさが m の上限を制約する（並列に保てる中間成果物が少なければ深く掘れない）。
**役割分担**：W＝方向、n＝広さ、m＝深さ。補正(α・Θ・Δ)はmに作用し、界面(I・O)は構造的制約として補正対象外。
**時間性**：C は固定値ではなく、IPOサイクルを通じて更新される状態を表す（C_t → C_t+1）。何がどう更新されるかは本編04・付録Aを参照。
**補助記法（O の二面）**：O_ext＝外に出す結果（他者が見る）／O_int＝内に残る結果（次サイクルの C に混ざる）。これは C の成分（状態変数）ではなく、Output 界面の二方向を区別する記法。定義と用法は本編04。

## 4. 接地と確信度（AIが重みを誤らないための注記）

各要素が既存研究のどこに足をつけ、どこまで検証されているか。
**【確立】**＝既存研究に接地／**【仮説】**＝検証可能だが未検証／**【思弁】**＝反証手段が未確立。

| 要素 | 接地（最も近い既存概念） | 確信度 |
| --- | --- | --- |
| I/P/O型・W | 目標志向性（mastery/performance）, 認知スタイル | 【仮説】 |
| n_p ↔ ワーキングメモリ | Miller 7±2, WM容量 | 【確立】 |
| n のベクトル分解（n_i/n_o独立） | 構成レベル理論, 知覚的分化 | 【仮説】 |
| m（処理の深さ） | 処理水準説（Craik & Lockhart） | 【仮説】 |
| α・Θ・Δ（補正3層） | 注意の脱落, 認知負荷, 信号検出 | 【思弁】 |
| Cの自己更新（経験で変わる） | 経験学習サイクル（Kolb）, ベイズ更新 | 【思弁】 |

> 注：このモデルは「形式化(記号がある)」されているが「操作化(測定手段がある)」されていない。
> 数式は導出された量ではなく、思考を整理するための座標系として読むこと。
