# §1 正典（定義元）

> このブロックがモデルの**単一の定義元**。他の節はここを参照するだけで、再定義しない。

## 1-1. 骨格

人のあらゆる認知を **Input → Process → Output** として捉え直す観察モデル。同じ I→P→O を見ても、どこに価値・興味を置くかが人によって違う、という出発点に立つ。

- **I型（Input注視）** … 情報・根拠・前提が揃っているかを気にする
- **P型（Process注視）** … なぜそうなるか・どう判断するかを気にする
- **O型（Output注視）** … 結局どうなったか・正解は何かを気にする

これは固定の「型分け」ではなく、W（注意配分）の偏りの典型例にすぎない。**W は人に固定された属性ではなく場面によって振れる**。同じ人が監査の場では I、企画の場では O に寄ることがある。「I型／P型／O型」は『その場面でその志向に寄っている状態』の略記。

## 1-2. 式

```
IPO × C

C = (W, n, m | α, Θ, Δ)
```

- 1行目 `IPO × C` … モデルの署名。**流れ（IPO）× 状態（C）** の掛け合わせ。IPO は「どこから・どの向きで読むか」、C は「いまの認知配置」。階層が異なるので行を分ける。
- 2行目 `C = (W, n, m | α, Θ, Δ)` … 状態 C の定義。縦棒の左（W, n, m）＝状態変数、右（α, Θ, Δ）＝補正パラメータ（すべて P 層）。

## 1-3. 変数表

| 記号 | 名称（AI言い換え） | 意味 | 構造 |
|---|---|---|---|
| W | 注意の向け先 | どこに注意を配分するか | ベクトル `(w_I, w_P, w_O)`, Σw=1 |
| n | 受け取り・並列・出力の細かさ | 界面と並列の解像度（横の広さ） | ベクトル `(n_i, n_p, n_o)` |
| m | 掘り下げの段数 | P の処理の深さ（縦の深さ・再帰段数） | スカラー（P 専属） |
| α | 補正が実行される確実さ | 補正がどれだけ確実に発火するか | スカラー（P 専属・m に作用） |
| Θ | 無意識に補正できる範囲 | 補正が自覚に上がる閾値 | スカラー（P 専属・m に作用） |
| Δ | 行間の補正力の上限 | 補正がどこまで可能か | スカラー（P 専属・m に作用） |

**n の内訳**：n_i = 受け取りの細かさ／n_p = 同時に保持できる処理の本数（≒ワーキングメモリ幅）／n_o = 出す細かさ。
**接続関係**：n_p の大きさが m の上限を制約する（並列に保てる中間成果物が少なければ深く掘れない）。
**役割分担**：W＝方向、n＝広さ、m＝深さ。補正（α・Θ・Δ）は m に作用し、界面（I・O）は構造的制約として補正対象外。
**時間性**：C は固定値ではなく、IPO サイクルを通じて更新される状態（C_t → C_t+1）。
**O の二面**：O_ext＝外に出す結果（他者が見る）／O_int＝内に残る結果（次サイクルの C に混ざる）。C の成分ではなく、Output 界面の二方向を区別する補助記法。

## 1-4. 確信度

**【確立】**＝既存研究に接地／**【仮説】**＝検証可能だが未検証／**【思弁】**＝反証手段が未確立。

| 要素 | 接地（最も近い既存概念） | 確信度 |
|---|---|---|
| I/P/O型・W | 目標志向性（mastery/performance）, 認知スタイル | 【仮説】 |
| n_p ↔ ワーキングメモリ | Miller 7±2, WM容量 | 【確立】 |
| n のベクトル分解（n_i/n_o 独立） | 構成レベル理論, 知覚的分化 | 【仮説】 |
| m（処理の深さ） | 処理水準説（Craik & Lockhart） | 【仮説】 |
| α・Θ・Δ（補正3層） | 注意の脱落, 認知負荷, 信号検出 | 【思弁】 |
| C の自己更新（経験で変わる） | 経験学習サイクル（Kolb）, ベイズ更新 | 【思弁】 |

> このモデルは「形式化（記号がある）」はされているが「操作化（測定手段がある）」はされていない。数式は思考を整理する座標系として読むこと。

## 1-5. 用語の言い換え表（AI 出力用）

| 正式名称 | AI の言い換え |
|---|---|
| I型 | Input注視型 |
| P型 | Process注視型 |
| O型 | Output注視型 |
| W | 注意の向け先 |
| n | 受け取り・並列・出力の細かさ |
| m | 掘り下げの段数 |
| α | 補正が実行される確実さ |
| Θ | 無意識に補正できる範囲 |
| Δ | 行間の補正力の上限 |

## 1-6. 構造と理論の縮尺（レイヤー注記）

- **構造（軸）**：「W・n・m という見る次元が在る」。何も予測しない。地図の経線にあたる。
- **リンク・法則**：「n_p が m を縛る」「補正はこう振る舞う」。外れうる。地図の*読み方*の主張。

理論的に最小の構造は **W・n・m の軸まで**。軸どうしが作用する（リンク）と言った時点で理論側に片足が入る。運用上は **α・Θ・Δ まで「コア」として束ねて扱う**――これは伝達可能性を制約に入れた設計判断であり、妥協ではなく決定。

浮きを防ぐ道具は一つ：各リンクについて**「外してみて、本当に必要な区別が消えるか？」**を問う最小性テスト。消えなければそのリンクは必要。消えなければ、それは構造のふりをして紛れ込んだ理論。

# §2 本編

> 全編を通じて **新人（O寄り）と先輩（P寄り）** をテストケースに使う。変数で迷ったらこの二人に戻す。定義はすべて §1 を参照。確信度タグ【確立】【仮説】【思弁】を尊重する。

## §2-1 入口：W（注意の向け先）

### ありふれたすれ違いから

> 新人「結局どうすればいいですか。答えを教えてください」
> 先輩「いや、ちょっと考えればわかるでしょ」

二人とも、相手のほうに問題があると感じて別れる。でも――これは能力の差でも、やる気の差でもないかもしれない。二人は同じ仕事の流れの、**違う場所を見ている**だけかもしれない。【仮説】

### 3つの注視

どんな仕事も判断も、おおまかに同じ形で流れる。

```
Input  →  Process  →  Output
（材料）  （処理）    （結果）
```

違うのは、この流れのどこに価値や興味を置くかだ。

| 型 | 見ている場所 | 口ぐせ |
|---|---|---|
| I型 | Input（情報・根拠・前提） | 「材料は揃ってる？根拠は？」 |
| P型 | Process（判断・仕組み・因果） | 「なぜそうなる？どう考える？」 |
| O型 | Output（結果・成果・正解） | 「結局どうなった？正解は？」 |

新人は **O**（結果）を見ている。先輩は **P**（過程）を見ている。どちらも間違っていない。見ている場所が違うだけだ。この「どこに注意を向けているか」を **W（注意の向け先）** と呼ぶ。【仮説】

## §2-2 個人差：n・m（広さと深さ）

### もう一種類のすれ違い

今度は**二人ともPを見ている**（同じW）。それでも噛み合わないことがある。

> 先輩「少し考えればわかるはずだよ」
> 新人「……その『少し』の中で、何が起きてるんですか？」

先輩の頭の中では A → B → C → D という思考の刻みが見えている。新人の中には A → D しかない。**間の刻みがそもそも存在しない。** これは能力でも怠慢でもなく、**ものの見方の細かさ**の差だ。この「細かさ」を **n（解像度）** と呼ぶ。【仮説】

### 解像度 n ― 横の広さ

n は一つの量ではなく、流れの3か所それぞれに分かれている。

| 成分 | 何の細かさか |
|---|---|
| n_i | 受け取りの細かさ（観察・傾聴の粒度） |
| n_p | 同時に保持できる処理の本数（≒ワーキングメモリ幅） |
| n_o | 出すときの細かさ（表現の粒度） |

「少し考えればわかる」のすれ違いは主に **n_p** の差だ。n_p の上限はワーキングメモリ容量（いわゆる 7±2）に対応する。【確立】

> **界面はシングルスレッド**：`I → P → O` のパイプライン自体は並列に流せない（「人はマルチタスクをしない」と言われる所以）。並列化できるのは P の内部だけで、その幅が n_p。だから WM 容量の限界が n_p の上限として現れる。n_i／n_o は「同時に何本走るか」ではなく「一本をどれだけ細かく刻むか」なので、n_p とは別軸になる。【確立：n_p ↔ WM容量】

**n_i と n_o は独立に異なりうる**（旧モデルが n_i = n_p = n_o を暗黙に仮定していた点の修正）。【仮説】

- n_i 高・n_o 低：細かく受け取り粗く出す（観察者・研究者タイプ）
- n_i 低・n_o 高：粗く受け取り細かく出す（表現者・職人タイプ）
- n_i 高・n_o 高：両方細かい（翻訳者・通訳タイプ）

### 深さ m ― 縦の掘り下げ

n が「横の広さ」なら、**m は「縦の深さ」**だ。「なぜ？」の答えがさらに「ではなぜ？」を生む。この掘り下げの段数が m。

```
m 小（浅い）：I → P → O
m 大（深い）：I → P → Pk → P' → … → O   ← 中間結果がまた入力に戻る
```

> **「→」もProcessである**：`I → P → O` の P の内部は、それ自体が `I → P1 → P2 → … → O` という多段構造をもつ。この刻みの段数こそが m。だから m は「Process 専属」になる。
> **Pk と I' は相の名前**：`Pk`（P 内部の中間成果物）と、それが次段の材料になった `I'`（再投入される入力）は、別の物ではなく**同じ実体の局面（相）違いの呼称**だ。この「中間成果物が入力へ転じる臨界点」を、本モデルは**相転移（処理のデジタルな切れ目）**として扱う（他の切れ目との並びは §5 B-1）。
> **連鎖は深さの外部展開**：O を新しい I として投げ直す「連鎖」は、深さ m が1ユニットに収まりきらず O 界面を経由して外に展開したものだ。内部で閉じれば深い処理、外（O_ext）を経由すれば連鎖に見える――同じ深化の、界面をまたぐか否かの差にすぎない。

**広さと深さは別物だが、つながってもいる**：並列に保てる本数が少なければ（n_p 小）、戻して掘り直す余裕がなく深く掘れない（m の上限が下がる）。【仮説】

**なぜルーティンだと学びが起きにくいか**：m が浅いまま固定されると掘り下げが起こらない。マニュアル化・ルーティン化は n_p を抑え、結果として m を浅く固定する。これが「定型業務では学習・創造が起きにくい」理由の構造的説明。【仮説】

> **m を入れると射程が変わる**：W・n だけなら本モデルは「認知タイプの記述」に留まる。そこに m（深さ）が加わると、学習（中間成果が新たな問いを生む）・問題解決（部分解がより深い問題を開く）・創造（断片が別の断片を誘発する）の**発生条件**を説明する側へ広がる。以降の §2-4（変化）・§2-5（創造）はこの拡張の上に乗っている。

## §2-3 補正：α・Θ・Δ（行間を埋める働き）

### 補正とは

人は会話のたびに無意識で**行間を埋める**。省略された刻みを補い、言われなかった前提を足し、相手の意図を推測する。この働きが補正だ。補正は Process の中で起きる。界面（I・O）は構造的制約であり補正の対象外。【思弁】

### 同じ説明が、3通りに分かれる

| 段階 | 体感 | 反応 |
|---|---|---|
| ① 快適 | 自動で埋まる | 「なるほど」（埋めた自覚すらない） |
| ② コスト増 | 頑張れば埋まる | 「わかる、けど疲れる」 |
| ③ 破綻 | 埋めきれない | フリーズ・誤読・回避 |

### 段階を決める3つのパラメータ

働く順番は **α → Θ → Δ**。

| 記号 | 役割 | 効く境目 |
|---|---|---|
| α | そもそも発火するか | 全段階の手前 |
| Θ | 自動で埋められる範囲（快適↔コストの境） | ①↔② |
| Δ | 埋められる力の上限（コスト↔破綻の境） | ②↔③ |

- **α（発火）**：疲れ・注意散漫・関心の低さで α が下がると、埋める前に読み飛ばす＝「うっかりミス」。能力ではなく*発火しなかった*だけ。
- **Θ（自動の範囲）**：ギャップが Θ 以内なら自動で埋まる。超えると意識的努力が要る「コスト」状態。
- **Δ（上限）**：意識的に頑張っても越えられない壁。超えると破綻。

【思弁】

### 「読みやすいのに読みにくい」の正体

読みにくく感じるとき、多くの場合それは Δ を超えている（理解不能）のではなく、**Θ を超えている（理解可能だがコストがかかる）＝段階②**だ。だから「読める」のに「疲れる」。【仮説】

この層が意味を持つのは、3段階が別々の観察可能なサインを持つときだけだ（①即時・無自覚 ／ ②遅延・自覚・疲労 ／ ③誤り・回避・フリーズ）。ここまで絞れば【仮説】、絞る前は【思弁】。

## §2-4 変化：C の自己更新

### 出力は2つある ― O_ext と O_int

1回のIPOは、外に出す結果だけを生むのではない。**自分の中に残る結果**も同時に生む。

| 出力 | 何か | 誰が見るか |
|---|---|---|
| O_ext | 提出物・発言・成果 | 他人 |
| O_int | 「こうすればよかった」「次はここを見よう」という澱 | 自分だけ |

O_int は次のサイクルの C に混ざる。だから C は回るたびに変わる。これが自己更新だ。【思弁】

```
C_t →（IPOを1回まわす）→ O_ext ＋ O_int → C_t+1
```

### 同じ経験でも、変化の向きは分かれる

**C を更新するのは、起きた事実（O_ext）ではなく、間に挟まる解釈（P）だ。**

同じ失敗をしても――

- 新人A「やっぱり自分には向いてない」→ W が**自己評価**へ向く
- 新人B「この手順の、ここが穴だ」→ W が**構造**へ向く

外で起きたこと（O_ext）は同じ。なのに残る澱（O_int）が違う。だから更新の向きが分岐する。【仮説】

「経験を積めば誰でも伸びる」は成り立たない。伸ばすのは O_ext の量ではなく、**間に挟まる P（解釈）の質**だ。

### 更新の波及

1. **n_p が空く**：繰り返すうちに要素が塊（チャンク）になり並列に保てる余裕が生まれる。【確立】
2. **m が深くなる**：余裕が空くとより深く掘れる（n_p → m の制約が時間方向にほどける）。
3. **Θ が広がる**：かつて段階②だったことが段階①になる。これが「楽になった」の正体。
4. **W が動く**：熟達すると注視点そのものがずれることがある（新人の O 型が先輩の P 型へ）。

逆に、O_int を残さなければ C は更新されない。ルーティン化で m が浅く固定されると、**回数だけ増えて中身は変わらない**。【仮説】

### 「個」とは何か

この構造をとるとき、「個」は固定属性の束ではない。**世界から Input を受け取り、Output を返しながら、自分自身を更新し続ける再帰システム**として定義される。これは同時に「なぜ人は変わるのか」の構造的説明になっている――変わるのは属性が書き換わるからではなく、回すたびに O_int が C に混ざり込むからだ。

## §2-5 起点と創造

### 流れは輪の一部

一つのIPOの結果（O）は次の材料（I）になる。だから流れに「始まり」も「終わり」もない。

```
… → O → I → P → O → I → P → O → …
```

この連続のどこかに線を引いて「ここが Input」と呼ぶ――その線の引き方が観測者の選択だ。【思弁】

### OPI ― 結果から読む

- 普通の向き（IPO）：材料を見て（I）→ 考えて（P）→ 答えを出す（O）
- 逆の向き（OPI）：まず「こういう結果がほしい」（O）から始め → そこへ至る過程を逆算し（P）→ 必要な材料を取りに行く（I）

OPIは別の能力ではない。**同じ流れを、違う起点から読んでいるだけ**だ。

起点が固定され、いつも材料（I）からしか入れないと、与えられた材料の組み合わせしか出てこない。起点を自由に選べると――欲しい結果（O）から、あるいは途中の処理（P）から――流れを組み直せる。これが「創造的」と呼ばれるものの一面だと読める。【思弁】

### 正直な位置づけ

「超高速で IPO を回している人」と「OPI で逆算している人」は、振る舞いの上で区別がつかない――しばしば本人にも区別できない。**反証可能性はほぼゼロ**だ。【思弁】

この節は「測れる主張」としてではなく、**ものの見方を一つ増やす装置**として置く。価値は予測力ではなく生成力の側にある。

### 本編の骨格まとめ

```
W       どこを見るか        （§2-1）
n・m    どれだけ細かく・深く （§2-2）
α・Θ・Δ どう行間を埋めるか   （§2-3）
更新     どう変わるか        （§2-4）
起点     どこから読むか      （§2-5）
```

C は認知状態の*記述*であると同時に、*操作できる対象*でもある――この二面性が本編の結論にあたる。

# §3 使い方

> 評価軸は「真か（妥当性）」ではなく「**使えるか（有効性）**」。ツールは真である必要はなく、使えればいい。

## §3-1 AIへの写像（プロンプト設計）

本編では C を「人がどう違うか」を*読む*ために使った。ここでは C を*書く*ために使う――自分や相手の C に合わせて「出力をどう設計するか」の設計図として。

### 写像表（C → プロンプト操作）

| 変数 | プロンプトでの動かし方 | 言い方の例 |
|---|---|---|
| W | 焦点を I／P／O のどこに置くか指定 | 「結論だけ（O）」「根拠を添えて（I）」「考え方の筋で（P）」 |
| n_i | 与える前提・情報量 | 「この資料だけを前提に」「背景は省いて」 |
| n_o | 出力の粒度 | 「3行で」「箇条書き5点で」「網羅的に」 |
| n_p | 同時に扱う観点の数 | 「観点は3つまで」「多面的に」 |
| m | 掘り下げの段数 | 「なぜを3段掘れ」「前提を疑え」／浅く：「一段で」 |
| α・Θ・Δ | 受け手の Θ に収める | 「素人にも自動でわかる粒度で」＝相手の Θ の内側へ |

### 同じ依頼を、相手の C で出し分ける

「新しい勤怠ルールを周知して」という一つの依頼を宛先で変える例：

- **O型の上司へ**（W=O, n_o小, m浅）：「結論から3行。何が・いつから・現場の手間が増えるか、だけ。」
- **P型の同僚へ**（W=P, m深め）：「なぜこの変更が必要か、旧ルールのどこが穴だったかの筋を含めて。」
- **I型の監査担当へ**（W=I, n_i大）：「根拠規程と適用範囲、例外条件を明示して。出典つきで。」

中身は同じ。**W・n・m の指定だけ**を変えている。

### 曖昧プロンプト → C 明示プロンプト

> ✗「この資料、いい感じにまとめて」
> ○「この資料を前提に（n_i）、結論を先頭3行で（W=O, n_o小）、根拠は各項1行添える（I）。掘り下げは一段で（m浅）。」

「いい感じ」は AI に自分の C を推測させている。明示すれば推測のブレが消える。

### よくある落とし穴

- **自分の C のまま書いてしまう**：書き手が高 n・高 m だと、指定しない限り AI も高 n・高 m で返し、受け手の Θ を超える。**宛先の C を指定する。**
- **W を指定しない**：放っておくと AI は結論を急ぐ O 寄りに寄りやすい。筋や根拠が欲しいなら P・I・m を明示。
- **m と n_o を混同する**：「詳しく」は n_o（出力を細かく）か m（深く掘る）か曖昧。分けて指定する。

### クロスAI運用

複数の AI を使い分けるとき、この写像表は**共通のものさし**になる。同じ C 指定のプロンプトを各 AI に投げ、出力差を見れば「どの AI が P を深く掘れるか」「O 要約はどれが安定か」をモデルの語彙で比較できる。

## §3-2 設計と運用

### 設計軸 ― 学びが起きる場をつくる

#### マニュアルは、たいてい O 層しかない

普通のマニュアルは手順（O）だけが書いてある。すると I型は「根拠は？」で止まり、P型は「なぜこの順番？」で止まる。3層を補えば、どの型にも入口ができる。

| 層 | 補う内容 | 効く相手 |
|---|---|---|
| I層 | 前提・材料・適用範囲・例外条件 | I型 |
| P層 | なぜこの手順か・判断の分岐点 | P型 |
| O層 | 手順そのもの（既存） | O型 |

#### 設計のジレンマ ― 全部書くと、学ばない

全部を spell out すると、読み手は行間を埋める必要がなくなる＝深さ m が発火しない＝学習が起きない（ルーティン化と同じ構造）。逆に足りなすぎると Δ を超えて破綻する。

```
全部書く（Θの内）    → 即できる、でも育たない
少し余白（Θの少し上）→ 自分で埋める、育つ      ← 狙い
余白すぎ（Δの外）    → 埋まらない、折れる
```

狙いは「**Θ の少し上、Δ の下**」。即戦力マニュアルは Θ の内（迷わせない）、育成教材は Θ の少し上（あえて考えさせる）。【仮説】

### 運用軸 ― 自分の C をその場で操作する

C は固定値の記述であると同時に、回せるツマミだ（§2-5）。状況に応じてどれを回すかを選ぶのが運用。

| 場面 | 上げる | 下げる |
|---|---|---|
| レビュー・監査 | m（掘り下げ）・n_i（読み込み） | スピード要求 |
| 提出・報告 | W=O（結論優先） | m・n_o（細かさを絞る） |
| 着想・企画 | α（探索の幅） | 正確さの要求 |

#### Temperature 論 ― α を意図的に振る

- **高温**（α のばらつき大）：発散・着想。突飛な接続を許す
- **低温**（α が安定）：収束・実行。毎回同じ品質を出す

**着想は高温 → 実行は低温**。AI の temperature と同じ発想で、人にもチームにも適用できる。【思弁】

**グローバルに最適な C は存在しない。** あるのは場面ごとに適切な C だけだ。「全部を70点で素早く回し、要所だけ深掘る」ほうが全項目満点を狙うより強いことが多い。【仮説】

### このモデルを「いつ」使うか

**2要素以上の差分を見るときに最も効く**。モデルの本領は「なぜ違うか」「どこがズレてるか」を記述すること。比較・関係・ズレを見たいときに糊として機能する。【仮説】

単一の対象でも、W で割る（I・P・O への重みを分解する）・n で割る（粒度の違いを見る）・束ねる（複数の観察を一軸に統合する）という使い方はできる。

# §4 付録A：検証可能な予測と検討課題

> 評価の方針：何でも説明できる主張は予測しない。ここに載るのは「**こうなれば外れる**」と言える形にできたものだけ。

## 検証可能な予測

**P1. n_i と n_o の二重乖離**（§2-2）
受け取りの細かさ（n_i）と出す細かさ（n_o）は独立に動く。
*検証*：聞き取り課題と表出課題の成績が個人内で乖離するか。観察者型（n_i高・n_o低）と職人型（n_i低・n_o高）が分離して見つかれば支持、相関一本に潰れれば反証。【仮説】

**P2. n_p が m の上限を縛る**（§2-2）
並列保持容量が小さいほど、深く掘れる段数の上限が下がる。
*検証*：ワーキングメモリ容量（n_p 代理）が低い人は深い再帰推論で頭打ちになるか。WM 負荷を外から上げると掘り下げ段数が落ちるか。【仮説】

**P3. ルーティン化 → m 浅化 → 学習停止**（§2-2・§2-4）
並列保持を許さない定型化は深化を止め、学習を起こさなくする。
*検証*：手順を全指定した条件と行間を残した条件で、同じ期間後の習熟・転移に差が出るか。【仮説】

**P4. 補正3段階の分離観察**（§2-3）
快適・コスト・破綻は一本の難易度尺度に潰れず、別々の閾値で切り替わる。
*検証*：同一人物で反応時間・主観疲労・誤り率が別々の点で切り替わるか。「②は平気だが③で崩れる人」が存在するか。【仮説】

**P5. α は能力から独立**（§2-3）
能力が高くても発火（α）が低ければミスは出る。
*検証*：能力指標を統制しても、注意状態（疲労・関心の操作）でミス率が独立に動くか。【仮説】

**P6. 「読みにくさ」＝Θ超え・Δ内**（§2-3）
理解可能だがコストがかかる帯域（Θ超え・Δ内）は、理解不能（Δ超え）と区別できる。
*検証*：同一テキストで、理解度は保たれるが処理時間・主観コストだけ上がる帯域が観測できるか。【仮説】

**P7. 同一 O_ext で O_int が分岐**（§2-4）
同じ結果でも、間に挟まる解釈（P）で更新方向が分かれる。
*検証*：同じ失敗フィードバック後、原因帰属（自己／構造）が分かれ、その後の行動変化が帰属によって予測できるか。【仮説】

**P8. O_int の形成が変化を分ける**（§2-4）
振り返り（O_int）の有無が、経験量が同じでも習熟差を生む。
*検証*：振り返りを促す介入のある／なしで、同一経験量でも習熟に差が出るか。【仮説】

## まず試すなら

コストが低く結果が診断的な順に：

1. **P1（二重乖離）** … 既存の聞き取り／説明課題で組める。型の実在を最短で検証できる。
2. **P4（段階分離）** … 反応時間＋疲労＋誤り率の3指標。補正層の核を直接突く。
3. **P7（O_int 分岐）** … 帰属の分かれと後続行動。変化の理論の中心。

この3つで支持が出れば、残り（P2/P3/P6/P8）の本格検証に進む価値がある。

## 検討課題（未解決の宿題）

- **操作的定義**：W・m・α・Θ・Δ に測定手段（代理指標）は作れるか。これが最大の宿題。
- **m の再定義**：いま m は「Process 内部の深さ（再帰段数）」だが、界面往復として読み直せないか。I/P 界面・P/O 界面・P/P' 界面（黙考・内省）の3種をまたぐ往復回数として統一すると n（広さ）と m（往復）が同じ土俵に乗る。ただし P/P' は外から見えず、測定窓は 2/3 に減る。
- **階層順序**：α → Θ → Δ の順序は固定か、入れ替わりうるか。

# §5 付録B：思弁の隔離

> ここには、面白いが**外から検証する手段がほぼない**発展アイデアを集める。同居させると、思弁の弱さを理由に本体の検証可能な部分まで否定されてしまうため、隔離する。これは品質の切り分けであって、価値の格下げではない。すべて【思弁】。

## B-1. クオリア（帰属としての扱い）

このモデルはおそらく、個人の**外界境界面**を表している――先に述べた社会的境界面（A↔B）のことだ。

「クオリア（主観的な感じ）」とは、観測者Bが、相手Aの境界面（I→P の知覚面・P→O の表出面・A↔B の社会的境界面）に現れた信号から**構成する帰属**である、とする見方。

扱うのは「帰属がどう発生するか」だけ。意識そのものがなぜ生じるか（ハードプロブレム）は扱わない。AIへのクオリア帰属も「帰属の発生機構が同じ」とは言うが「だから本当に感じている」とは主張しない。

### 境界面の存在論的地位 ― 3つのデジタルな切れ目

3つの変換面のうち、**I→P（知覚面）と P→O（表出面）は、連続した流れに観測者が引いた格子の線**にすぎない。どこを「入口」「出口」と呼ぶかは観測の取り方の問題だ（§2-5 の起点自由度と同じ話）。

ところが **A↔B の社会的境界面だけは、観測者が引いた線ではなく物理的に実在する分断**である――二つの認知系は端的に別個の系だからだ。ここに、本モデルが「アナログな連続」ではなく「デジタルな切れ目」として扱う点が3つ揃う：

1. **相転移**（§2-2）… 中間成果物が入力へ転じる臨界点
2. **Θ越え**（§2-3）… 快適から意識的コストへ切り替わる境
3. **個体間境界**（本節）… A と B を分ける物理的な分断

前二者は「流れの中の切れ目」だが、三つ目だけは系そのものの切れ目である点で性質が異なる。クオリア帰属が発生するのは、まさにこの実在する分断を越えて B が A の内側を逆推定するからだ。【思弁】

**正直な評価**：他我問題・志向姿勢の再記述として妥当だが新規ではない。「ゾンビ問題の構造的言い換え」は言い過ぎ――ゾンビ論証の前提を受け入れているだけで、問題を解いてはいない。本体から完全に切り離し、独立の思弁として置く。【思弁】

## B-2. Human OS（認知カーネル説）

個人の外界境界面（B-1）が実在するなら、人の認知はOSのような層構造として捉えられるかもしれない。ハードウェア層（身体・脳）／認知カーネル層（＝C にあたる中核）／感情層／社会層／アプリ層のように積み、層と層は補正パラメータ（α・Θ・Δ）で連絡するという構想。

**正直な評価**：整理メタファーとしては有用。だが「層が α・Θ・Δ で連絡する」は検証手段がなく現状は願望に近い。層の数や境界に必然性の根拠もない。IT に馴染んだ読み手の Θ を下げる言い換え道具としては持てる。ここから予測を引き出さない。【思弁】

## B-3. 多層への拡張と境界面の交差（着想メモ）

B-2 の層構造を受けて、思いつきとして置いておく。

```
IPO × A × B × C
A = (W, n, m | α, Θ, Δ | g1, g2, g3)
B = (W, n, m | α, Θ, Δ | g1, g2, g3)
C = (W, n, m | α, Θ, Δ | g1, g2, g3)
```

A は「記憶層」、B は「感情層」、C は「知覚層」――これまでの議論は全部を C にまとめて知覚層として扱ってきたのではないか、という問い。g は次元軸の重なり要素を表す想定。

さらに、これらの層を持つ**複数の個人**が接したとき、何が生まれるか、という着想：

- **AとBの境界面が接した場合 → 面ができる。** このとき面に生まれるのは何か？　それを**心**というのではないか？
- **AとBとCの境界面が接した場合 → 線ができる。** このとき線に生まれるのは何か？　それを**自己**というのではないか？
- **AとBとCとDの境界面が接した場合 → 点ができる。** このとき点に生まれるのは何か？　それを**アイデンティティ**というのではないか？

2者の接触が面（広がり）を生み、3者で線（輪郭）に絞られ、4者で点（唯一性）に収束する――という幾何学的な直観。検証手段は現状ない。【思弁】

### B-4-0. 設定

時刻 `t` は離散。1ステップ ＝ 1回の IPO サイクル。状態 `C_t ∈ 𝒞` が各ステップで更新される。

```
C = (W, n, m, α, Θ, Δ)

W = (w_I, w_P, w_O) ∈ Δ²        w_I + w_P + w_O = 1,  w ≥ 0
n = (n_i, n_p, n_o) ∈ ℝ³_{>0}
m ∈ ℝ_{≥0}
α ∈ [0,1]
0 ≤ Θ ≤ Δ

状態空間:
𝒞 = Δ² × ℝ³_{>0} × ℝ_{≥0} × [0,1] × { (Θ,Δ) | 0 ≤ Θ ≤ Δ }

構造制約:  m ≤ φ(n_p),   φ は単調増加        （n_p が m の上限を縛る）
```

- 縦棒の左右（状態変数 W,n,m ／ 補正 α,Θ,Δ）は直積の前半・後半に対応。
- `Θ ≤ Δ` を空間に埋め込むことで「快適の境は破綻の境より手前」が定義から保証される。
- `W ∈ Δ²`（2-単体）により `Σw=1` が自動で効く。

---

### B-4-1. 1サイクル ＝ 4作用素の合成 `Φ_C`

外界刺激 `s` から外部出力 `O_ext` までを、Cで変調された合成写像で書く。

```
Φ_C = Write ∘ Process^m ∘ Fill ∘ Read

O_ext = Φ_C(s)
```

#### ① Read（注意による読み出し）― W, n_i

刺激を I/P/O 三チャネルに分け、W で重みづけ、解像度 n_i で量子化する。

```
x = Read(s; W, n_i) = ( w_I · q_{n_i}(s_I),
                        w_P · q_{n_i}(s_P),
                        w_O · q_{n_i}(s_O) )
```

`q_n` ＝ 解像度 n の量子化作用素（n 大ほど細かく刻む）。

#### ② Fill（行間補正）― α, Θ, Δ

x の欠損量を `g(x)` とする。発火 `A ~ Bernoulli(α)` で分岐。

```
        ┌ x              if A = 0            未発火 → 欠損残存（うっかりミス）
x̃ = Fill(x) =
        │ complete(x)    if A = 1, g ≤ Θ     ①自動充填（コスト 0）
        │ complete(x)    if A = 1, Θ < g ≤ Δ ②充填（コスト κ·(g−Θ)）
        └ ⊥              if A = 1, g > Δ     ③破綻

払ったコスト:  k(x) = max( 0, κ · ( min(g, Δ) − Θ ) )
```

理解度と充填コストは同じ `g` の別関数として分離される：

```
U(g) = 1   if g ≤ Δ   else 0      （理解できるか）
K(g) = k   （上式）               （どれだけ疲れるか）
```

→ `Θ < g ≤ Δ` の帯域は「理解できる（U=1）のに疲れる（K>0）」を表す。

#### ③ Process^m（深さ m の再帰）― n_p

処理を再帰写像にし、中間結果を入力へ戻す。並列保持の上限が n_p。

```
z_0   = x̃
z_{j+1} = Process(z_j; n_p),     j = 0 … m−1
処理結果 = z_m

掘り続ける条件:  Σ_{j} k(z_j) ≤ B(Θ, Δ)      補正コスト予算 B を超えたら止まる
```

この不等式が三者を同じループで縛る：

- `n_p → m`（構造制約 `m ≤ φ(n_p)`）… 並列に保てる本数が深さの上限を決める
- `Θ, Δ → 実効深さ`（コスト予算 B）… 補正コストを使い切ると掘れない

#### ④ Write（出力）― W, n_o

```
O_ext = Write(z_m; W, n_o) = q_{n_o}(z_m)
```

---

### B-4-2. 内部出力と更新則 ― ループを閉じる

外に出すのと同時に、内に澱が残る。澱は **結果（O_ext）ではなく処理の軌跡** から生まれる。

```
trace_t = (z_0, z_1, …, z_m)
cost_t  = Σ_{j} k(z_j)

O_int_t = ψ( trace_t, cost_t )        ← O_ext には依存しない
```

これが次の状態を作る：

```
C_{t+1} = C_t ⊕ η · U(O_int_t)

成分ごとの具体形:
  n_p   ← n_p + δ · chunk(t)                   チャンク化で並列余裕が空く
  m上限 ← φ(n_p)                                n_p↑ に伴い自動で上がる
  Θ     ← Θ + λ · { かつて k>0 だった領域 }      ②が①に降りる ＝「楽になった」
  W     ← Π_{Δ²}( W + μ · w(O_int_t) )          注視点の移動（単体へ射影）
  α, Δ  ← 状態依存の微調整
```

- `Π_{Δ²}` ＝ 2-単体への射影（更新後も `Σw=1` を保つ）。
- 「同じ O_ext でも更新が分岐する」は、ψ が `trace`（＝P の質）に依存し O_ext に依存しないことで表現される。
- 「回数だけ増えて中身が変わらない」は、`m` が浅く固定され `trace` が薄い → `O_int_t ≈ 0` → `C_{t+1} ≈ C_t` として出る。

---

### B-4-3. 輪として閉じる ― 起点も終点もない流れ

出力が次の入力の一部になる。

```
s_{t+1} ⊇ O_ext,t          外界に出した結果が次の材料に混ざる
```

全体は、状態 `C_t` をパラメータに持つ非自律な離散力学系：

```
( s_{t+1}, C_{t+1} ) = G( s_t, C_t )

  O_ext,t  = Φ_{C_t}(s_t)
  C_{t+1}  = C_t ⊕ η · U( ψ(trace_t, cost_t) )
  s_{t+1} ⊇ O_ext,t
```

---

### B-4-4. 起点自由度 ― IPO / OPI を同じ写像の順・逆として

```
IPO（順問題）:  O_ext = Φ_C(s)         s を固定 → O を得る
OPI（逆問題）:  s ∈ Φ_C⁻¹(O*)          O* を固定 → s を逆算
```

`Φ_C` は Read で次元を落とし、Fill で `⊥` を作り、量子化で潰すため一般に **非単射**。
よって `Φ_C⁻¹(O*)` は集合（複数解、または空）。

- 起点の選択 ＝ 順方向 `Φ_C` を回すか、逆方向 `Φ_C⁻¹` を解くかの選択。
- 「創造」＝ 逆像の集合 `Φ_C⁻¹(O*)` から一点を選ぶ自由度。
- 起点を I に固定して入る ＝ 順問題しか解かない ＝ 与えられた材料の組み合わせしか出ない。

---

### B-4-5. 全体署名（最小要約）

```
状態:    C = (W, n, m, α, Θ, Δ) ∈ 𝒞,   m ≤ φ(n_p),   0 ≤ Θ ≤ Δ
1サイクル: O_ext = (Write ∘ Process^m ∘ Fill ∘ Read)(s) = Φ_C(s)
学習:    C_{t+1} = C_t ⊕ η · U(ψ(trace_t, cost_t))
輪:      s_{t+1} ⊇ O_ext,t
創造:    s ∈ Φ_C⁻¹(O*)
```

W＝Read/Write のチャネル重み、n＝各作用素の量子化解像度、m＝Process の再帰段数、
α・Θ・Δ＝Fill の発火・自動範囲・上限。更新は C の漸化式、起点は順／逆の選択。
これで本編の全要素（§2-1 W ／ §2-2 n,m ／ §2-3 補正 ／ §2-4 更新 ／ §2-5 起点）が
一本の `Φ_C` とその力学系に閉じる。

> 注：κ, φ, ψ, η, λ, μ, δ, B の **関数形は決まっているが値は未操作化**。
> §4 の宿題（W・m・α・Θ・Δ の代理指標）が片付くまで、本ノートは
> 「測れる関数」ではなく「主張を曖昧さなく書くための座標系」として読む。

# §6 付録C：発展の記録（出発点との差分）

> ここは定義でも予測でもなく、**このモデルがどこから来たか**の記録。現行の定義は §1 が持つ。ここに残すのは「なぜ今の形になったか」――出発点からの差分と、その過程で得た気づき（とくに、モデルの形が作者自身の C を映していたという発見）である。本モデルの出発点は「IPO認知モデル仮説_加筆版-4」。

## C-1. 主な再定義

**n の再定義（単一スカラー → ベクトル）**
当初、n は「Process の分解数」という**単一のスカラー**として置かれていた。これはモデルの作者が P 型で、P の解像度だけを高く見ていたことによる**射影**だった――つまり *モデルの形が作者の C を反映していた*。この気づきが、n を I・P・O それぞれに解像度を持つベクトル `(n_i, n_p, n_o)` へ再定義する契機になり（§1-3・§2-2）、さらに「全変数が I/P/O の3軸成分を持ちうる」という総括（W・n はベクトル、m・補正はスカラー）へつながった。旧定義の「分解数」は、現行では n_p と、m へ移した深度に相当する。

**m の再定義（連鎖数 → 深さ）**
当初 m は「IPO ユニットの連鎖数」（横の繰り返し）だった。n を「横の広さ」と定めたのに合わせ、m は「縦の深さ」へ整理された。かつての連鎖は、深さが O 界面を経由して外へ展開したもの（§2-2 の「連鎖は深さの外部展開」）として本体に統合された。

**補正層の分離**
状態変数（W・n・m）と補正パラメータ（α・Θ・Δ）を、同列に並べるのをやめ**階層として分離**した。これにより式は `C = (W, n, m | α, Θ, Δ)` の縦棒つき表記になり、さらに上位に署名 `IPO × C` を置いた（§1-2）。

**時間性の明示と補助記法**
C を固定特性ではなく更新され続ける状態と見る `C_t → C_t+1` を明示し、Output 界面の二方向を分ける補助記法 `O_ext`／`O_int` を追加した（§1-3・§2-4）。この2つが後の「自己更新」と「同じ経験でも変化が分岐する」議論の土台になった。

## C-2. あえて削除した枝

- **学習スタイル論との優劣比較**：本筋（W・n・m の座標を立てること）に不要と判断し外した。
- **哲学的ゾンビ問題への接続**：ゾンビ論証の前提を受け入れているだけで、解決でも本質的言い換えでもないため外した。クオリア関連は「帰属」としてのみ残し、付録B（§5 B-1）に隔離した。

> この記録の意味：モデルの各再定義は「正しい理論に近づいた」というより「作者の C の偏りを一つずつ剥がした」過程に近い。だから発展の記録は、そのままモデルの自己適用（作者の W・n が定義に射影していた、という観察）の記録でもある。
