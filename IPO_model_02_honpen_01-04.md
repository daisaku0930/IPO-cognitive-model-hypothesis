# 本編 ― IPO認知モデル

> この本編は読み物の背骨。定義は**正典**を参照し、ここでは再定義しない。
> 全編を通じて **新人（O寄り）と先輩（P寄り）** をテストケースに使い、抽象的な変数が出たら必ずこの二人に戻して確かめる。原則は一度に一つずつ導入する。

---

# 01 ― 入口：ありふれたすれ違い

## どこでもよくある、ありふれたすれ違いから

> 新人「結局どうすればいいですか。答えを教えてください」
> 先輩「いや、ちょっと考えればわかるでしょ」

二人とも、相手のほうに問題があると感じて別れる。
新人は「不親切な人だ」と思い、先輩は「考えようとしない人だ」と思う。

でも――これは能力の差でも、やる気の差でもないかもしれない。
二人は同じ仕事の流れの、**違う場所を見ている**だけかもしれない。【仮説】

## その「流れ」には名前がある ― IPO

どんな仕事も判断も、おおまかに同じ形で流れる。

```
Input  →  Process  →  Output
（材料）  （処理）    （結果）
```

材料を受け取り（I）、頭の中で処理し（P）、結果を出す（O）。
誰もがこの流れを通る。違うのは、**この流れのどこに価値や興味を置くか**だ。

## 3つの注視

| 型 | 見ている場所 | 口ぐせ |
| --- | --- | --- |
| I型 | Input（情報・根拠・前提） | 「材料は揃ってる？根拠は？」 |
| P型 | Process（判断・仕組み・因果） | 「なぜそうなる？どう考える？」 |
| O型 | Output（結果・成果・正解） | 「結局どうなった？正解は？」 |

さっきのすれ違いに戻すと、構図はこうだ。

- 新人は **O**（結果）を見ている →「答えがほしい」
- 先輩は **P**（過程）を見ている →「過程を通れば答えは出るはず」

どちらも間違っていない。見ている場所が違うだけだ。
この「どこに注意を向けているか」を、モデルでは **W（注意の向け先）** と呼ぶ。【仮説】

# 02 ― 認知の個人差：広さと深さ

## もう一種類のよくあるすれ違い

今度は**二人ともProcessを見ている**（同じW）。それでも噛み合わないことがある。

> 先輩「少し考えればわかるはずだよ」
> 新人「……その『少し』の中で、何が起きてるんですか？」

先輩の頭の中では A → B → C → D という思考の刻みが見えている。
新人の中には A → D しかない。**間の刻みがそもそも存在しない。**

これは能力でも怠慢でもなく、**ものの見方の細かさ**の差だ（向け先 W が同じでも、これが違うと噛み合わない）。【仮説】
この「細かさ」を、モデルでは **n（解像度）** と呼ぶ。

## 解像度 n ― 横の「広さ」

n は一つの量ではなく、流れの3か所それぞれに分かれている。

| 成分 | 何の細かさか | 例 |
| --- | --- | --- |
| n_i | 受け取りの細かさ（観察・傾聴の粒度） | 同じ会議で拾う情報量の差 |
| n_p | 同時に保持できる処理の本数（≒ワーキングメモリの幅） | 一度に追える筋の数 |
| n_o | 出すときの細かさ（表現の粒度） | 説明の作り込みの差 |

「少し考えればわかる」のすれ違いは、主に **n_p** の差だ。
並列に保てる本数が多い人（n_p大）には自明な刻みが、少ない人（n_p小）の中には立ち上がらない。

> n_p の上限がワーキングメモリ容量（いわゆる 7±2）に対応する点は、既存研究に接地している。【確立】

### n_i と n_o は別々でありうる

受け取りの細かさと、出すときの細かさは、独立に動きうる。【仮説】

- n_i 高・n_o 低 … 細かく受け取るが粗く出す → 観察者・研究者タイプ
- n_i 低・n_o 高 … 粗く受け取るが細かく出す → 表現者・職人タイプ
- n_i 高・n_o 高 … 両方細かい → 翻訳者・通訳タイプ

> 「細かく受け取る人は細かく出すはず」という思い込みへの反例。二重乖離として検証可能。

## 深さ m ― 縦の「掘り下げ」

n が「横の広さ」なら、**m は「縦の深さ」**だ。
「なぜ？」の答えが、さらに「ではなぜ？」を生む。この掘り下げの段数が m。

```
m 小（浅い）：I → P → O
m 大（深い）：I → P → Pk → P' → … → O   ← 中間結果がまた入力に戻る
```

通常の処理は浅く通り抜ける（m小）。深いときは、途中で出た中間結果がもう一度入力として戻り、処理が処理を掘っていく。

**広さと深さは別物だが、つながってもいる**：並列に保てる本数が少なければ（n_p小）、戻して掘り直す余裕がなく、深く掘れない（m の上限が下がる）。【仮説】

### なぜルーティンだと学びが起きにくいのか

m が浅いまま固定されると、掘り下げ＝深化が起こらない。
マニュアル化やルーティン化は、しばしば並列保持を許さず（n_p を抑え）、結果として m を浅く固定する。
これが「**なぜ定型業務では学習や創造が起きにくいか**」の構造的な説明になる。【仮説】

---

## ここまでを束ねる座標系

ここまでで、人は **向け先（W）・広さ（n）・深さ（m）** の3つで違う、と見えてきた。
これらをまとめて扱う座標が C であり、流れ（IPO）と掛け合わせた全体像が、本モデルの背骨である。

```
IPO × C

C = (W, n, m | α, Θ, Δ)
```

1行目がモデルの署名（流れ × 状態）、2行目が状態 C の定義。
W・n・m はここまでに登場した。縦棒の右側（α・Θ・Δ）はこの先で扱う。各変数の定義は**正典**を参照。

> ここで一つ自覚しておく。**この文書自体が、書き手の高い n・m を反映した出力だ。**
> だから n・m が近い人には素直に入り、遠い人には「読みやすいのに読みにくい」と感じられる。その「読みにくさ」の正体こそ、次節（03）のテーマになる。

そして次の疑問が立つ。これだけ違う二人が、それでもなぜ「**だいたい伝わる**」のか？ そして、なぜ時々「**まったく伝わらない**」のか？

# 03 ― 補正：なぜ伝わる／伝わらないのか

答えは、人が会話のたびに無意識でやっている一つの働きにある。**補正** ――明示されていない行間を、自分で埋める働きだ。

## 補正とは ― 行間を埋める働き

先輩が「少し考えればわかる」と言ったとき、多くの場面で新人も*ある程度は*行間を自力で埋めている。
省略された刻みを補い、言われなかった前提を足し、相手の意図を推測する。この**埋める働き**が補正だ。

補正はProcessの中で起きる。【思弁】
（受け取り口・出し口そのもの＝界面（I・O）は構造的な制約であって、補正の対象ではない、という整理。）

## 同じ説明が、3通りに分かれる

新人がまったく同じ説明を受けても、結果は3つに分かれる。

| 段階 | 体感 | 新人の反応 |
| --- | --- | --- |
| ① 快適 | 自動で埋まる | 「なるほど」（埋めた自覚すらない） |
| ② コスト増 | 頑張れば埋まる | 「わかる、けど疲れる」 |
| ③ 破綻 | 埋めきれない | 「……まったくわからない」（フリーズ・誤読・回避） |

「伝わる／伝わらない」は0か1ではなく、この**3段階のどこに着地するか**の問題だ。【思弁】

## 段階を決める3つのツマミ

どの段階に着地するかは、3つのパラメータで決まる。すべてP層の働きで、深さ **m** に作用する。

| 記号 | 言い換え | 役割 | 効く境目 |
| --- | --- | --- | --- |
| α | 補正が実行される確実さ | そもそも発火するか | 全段階の手前 |
| Θ | 無意識に埋められる範囲 | 快適 ↔ コスト | ①↔② |
| Δ | 埋められる力の上限 | コスト ↔ 破綻 | ②↔③ |

働く順番は **α → Θ → Δ**。

- **α（発火）**：疲れ・注意散漫・関心の低さで α が下がると、埋める前に読み飛ばす。これが「うっかりミス」「読み落とし」。能力ではなく*発火しなかった*だけ。
- **Θ（自動の範囲）**：ギャップが Θ 以内なら自動で埋まり「快適」。超えると、埋まるけれど意識的な努力が要る「コスト」状態へ。
- **Δ（上限）**：意識的に頑張っても越えられない壁。ここを超えると埋めきれず「破綻」。

【思弁】

## 「読みやすいのに読みにくい」の正体

02 の最後に置いた予告の回収。

この文書を読みにくく感じるとき、それは多くの場合 **Δ を超えている（理解不能）のではなく、Θ を超えている（理解はできるが意識的コストがかかる）** 状態だ。つまり**段階②**。
だから「読める」のに「疲れる」。【仮説】

ここから設計の狙いが逆算できる。**Θ を超えさせないこと**――自動で埋まる粒度まで刻んでおくこと。
新人・先輩のテストケース、図解、言い換え表は、すべて読み手の Θ の内側に収めるための工夫だった。

## この層は最も思弁的 ― 正直な但し書き

α・Θ・Δ には、まだ測定手段がない。
そのため、どんな失敗も後から「α が下がった」「Δ を超えた」で説明できてしまう危険がある。**何でも説明できる説明は、何も予測しない。**

だからこのモデルがこの層で意味を持つのは、**3段階が別々の観察可能なサインを持つとき**だけだ。

- ① 快適 ＝ 即時・無自覚
- ② コスト ＝ 遅延・自覚・疲労
- ③ 破綻 ＝ 誤り・回避・フリーズ

この3つが一本の「難しさ」尺度に潰れず、人によって**分離して**観察できるなら――たとえば「②は平気だが③で崩れる人」「能力は高いが α だけ低くミスする人」が区別できるなら――それは検証可能な主張になる。ここまで絞れば【仮説】、絞る前は【思弁】。

> この条件付き【仮説】を検証の形にしたものが付録A・P4（補正3段階の分離観察）。

# 04 ― 変化：新人はなぜ先輩になるのか

ここまでは「**ある時点の** C」の話だった。だが新人は、やがて先輩になる。かつて「わかるけど疲れる」（段階②）だった説明が、いつのまにか「なるほど」（段階①）で済むようになる。
つまり **C は変わる**。なぜ、どのように変わるのか。

## 出力は2つある ― O_ext と O_int

1回のIPOは、外に出す結果だけを生むのではない。**自分の中に残る結果**も同時に生む。

| 出力 | 何か | 誰が見るか |
| --- | --- | --- |
| O_ext | 提出物・発言・成果 | 他人 |
| O_int | 「こうすればよかった」「次はここを見よう」という澱（おり） | 自分だけ |

そして **O_int は、次のサイクルの C に混ざる**。
だから C は、回るたびに少しずつ変わる。これが自己更新だ。【思弁】

```
C_t →（IPOを1回まわす）→ O_ext ＋ O_int → C_t+1
```

## 同じ経験でも、変化の向きは分かれる

ここが肝心なところ。
**C を更新するのは、起きた事実（O_ext）そのものではない。間に挟まる解釈（P）だ。**

同じ失敗をしても――

- 新人A「やっぱり自分には向いてない」 → W が**自己評価**へ向く
- 新人B「この手順の、ここが穴だ」 → W が**構造**へ向く

外で起きたこと（O_ext）は同じ。なのに残る澱（O_int）が違う。だから更新の向きが分岐する。【仮説】

ここから、よく聞く通念が崩れる。
「経験を積めば誰でも伸びる」は成り立たない。伸ばすのは **O_ext の量**ではなく、**間に挟まる P（解釈）の質**だ。

## 何が、どう変わるのか

更新は、おもにこう波及する。

1. **n_p が空く**：繰り返すうちにバラバラの要素が塊（チャンク）になり、並列に保てる本数の余裕が生まれる。【確立】
2. **m が深くなる**：余裕が空くと、より深く掘れる。――02 で見た「n_p が m の上限を縛る」関係が、*時間方向にほどけていく*。
3. **Θ が広がる**：かつて段階②（コスト）だったことが、段階①（快適）になる。これが「楽になった」の正体。
4. **W が動く**：熟達すると、注視点そのものがずれることがある（新人のO型が、先輩のP型へ）。

これが「新人が先輩になる」の中身だ。新人の「わかるけど疲れる」が、いつのまにか自動になる。

## 「変わらない」ことも、同じ仕組みで説明できる

逆を考えるとモデルの輪郭がはっきりする。

サイクルが O_ext だけを出して O_int を残さなければ、C は更新されない。
ルーティン化で m が浅く固定されると（02）、澱が溜まらず、**回数だけ増えて中身は変わらない**。

「10年やっても変わらない人」と「3年で化ける人」の差は、年数ではない。**O_int が形成されているか**だ。【仮説】

> 正直な但し書き：これも事後的には何でも説明できてしまう。検証可能になるのは、**同じ O_ext から異なる O_int が分離して観察できるとき**（上の新人A／Bが見分けられるとき）だけ。

# 本編ここまでの座標系まとめ

各節は、座標 `C = (W, n, m | α, Θ, Δ)` のどの成分を扱ってきたか。

| 節 | 扱ったC成分 | 一言でいうと |
| --- | --- | --- |
| 01 入口 | W | 人は流れのどこを見るかが違う |
| 02 個人差 | n（n_i/n_p/n_o）・m | 見方の広さと深さも違う |
| 03 補正 | α・Θ・Δ | 違いを越えて伝わる／伝わらないを分ける働き |
| 04 変化 | C 全体の更新 | その C が経験で書き換わる |

ここまでは、流れ（IPO）を必ず **I → P → O** の向きで読んできた。
だが――そもそもこの流れを「**どこから読み始めるか**」は、決まっていない。
同じ流れを別の起点から読むと何が起きるのか。それが次節（**05・起点と創造**）のテーマで、そこで本編が閉じる。

> 定義はすべて**正典**を参照。本編はモデルの中身を*説明する*もので、変数を*定義する*のは正典の役割。
